Fase de planificación y análisis (mayo 2025):

  • Reuniones iniciales, identificación de funcionalidades clave, selección de herramientas, y asignación de roles.
  • También incluye análisis técnico como verificación de hardware y selección de frameworks, lo que garantiza que el equipo esté preparado antes de codificar.

Fase de diseño (finales de mayo): Se trabajan los niveles, interfaces gráficas y elementos visuales, definiendo cómo se verá y comportará el juego. Fase de desarrollo e implementación (junio): Aquí se programa la lógica del juego, incluyendo físicas, colisiones, sistema de puntuación, integración de arte y Unity. Se ejecutan pruebas unitarias y funcionales, asegurando que cada módulo funciona correctamente. Fase de ajustes y pruebas finales (finales de junio): Se corrigen errores, se ajusta el juego, y se prepara el paquete final para distribución. Fase de entrega y post-lanzamiento (julio): Se lanza el juego, se brinda soporte al usuario, se corrigen bugs post-lanzamiento, y se evalúan mejoras y expansiones. Termina con una retroalimentación para el aprendizaje del equipo.
